# Journal Memory Workflow

> Goal: make daily notes a high-signal input source instead of a流水账 archive.

## Core principle

Daily notes should record what matters. Weekly review should extract patterns, decisions, and reusable insights.

## Daily note structure

- Key events
- Mood and state
- Work and learning progress
- Important spending
- Thoughts triggered by the day
- Items that may deserve later deposit
- The single most important thing for tomorrow

## Weekly review structure

- Important events of the week
- Emotion and energy pattern
- Progress and blockers
- Learning and AI progress
- Spending and family pressure
- Important insights
- Suggestions for `06_知识编译`
- Suggestions for `Agents/子木`
- Items that should stay only in the diary

## Deposit rules

- Ordinary facts stay in the diary.
- Methods, frameworks, and reusable experience go to the structured knowledge base.
- Personal goals, preferences, and decision principles go to the personal agent workspace.
- The raw memory layers stay read-only unless a system-level change is requested.

## Trigger phrase

```text
Please review this week's diary according to the Journal Memory Workflow and identify what should be deposited into the structured knowledge base or the personal agent workspace.
```
